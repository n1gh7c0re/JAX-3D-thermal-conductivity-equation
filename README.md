# [JAX-2-27] Уравнение теплопроводности 3D


## Background

Трехмерное моделирование теплопереноса необходимо для анализа терморегуляции электроники, процессов затвердевания материалов и климатического моделирования. 
Переход к 3D кратно увеличивает вычислительную сложность, делая актуальным применение физически-информированных нейросетей (PINNs) и GPU-ускоренных сеточных методов (JAX/CuPy) 
для преодоления «проклятия размерности».

## Task
Реализовать на JAX алгоритм решения нестационарного уравнения теплопроводности в трехмерной области.

Необходимо провести сравнительный анализ двух подходов:
1. Классический сеточный метод (FDM/FEM) с реализацией векторизованных операций в JAX (jax.lax.scan для времени)
2. Предложить архитектуру Physics-Informed Neural Networks (PINN), где решение аппроксимируется нейросетью , минимизирующей невязку уравнения в точках коллокации

## Validation & Comparison (Part 3)

Полная научная валидация обоих методов на знаменитом аналитическом тесте с известным точным решением.

### Структура

```
src/
├── analytical_solution.py    # Аналитическое решение (JIT-совместимо)
└── validation.py             # Все функции валидации, метрик, графиков

experiments/
├── run_fdm_validation.py     # Запуск FDM (готово)
├── run_pinn_validation.py    # Запуск PINN (готово)
└── run_full_validation.py    # ГЛАВНЫЙ СКРИПТ ВАЛИДАЦИИ

docs/
└── validation_report.md      # Научный отчет с таблицами и выводами

results/
├── fdm/
│   ├── fdm_u_num_N41.npy     # Численное решение FDM
│   ├── fdm_u_exact_N41.npy   # Точное решение на сетке
│   ├── metrics.csv            # Таблица сходимости
│   └── fdm_slice_t_0_1.png
├── pinn/
```

### Запуск валидации

```bash
# Сначала убедитесь, что результаты FDM и PINN вычислены:
python experiments/run_fdm_validation.py
python experiments/run_pinn_validation.py

# Затем запустите ПОЛНУЮ ВАЛИДАЦИЮ:
python -m experiments.run_full_validation --grid_size 41 --alpha 0.1 --t_max 1.0

# С опциями:
python -m experiments.run_full_validation --help
```

### Ключевые метрики

| Метрика | FDM (N=41) | PINN (21³) |
|---------|-----------|-----------|
| Точность (L²) | $O(10^{-4})$ | $O(10^{-2})$ |
| Время обучения | ~2 сек | ~150 сек |
| Время инференса | ~10 мс | ~1 мс |

### Основные выводы

1. **FDM** показывает превосходную точность для гладких решений на регулярной сетке: вторая ошибка сходимости бесспорна
2. **PINN** обеспечивает гибкость и может обрабатывать нерегулярные геометрии, но требует значительных вычислительных затрат на обучение
3. **Гибридные подходы** могут сочетать преимущества обоих методов
